# Prompts

One-off review, audit, and testing prompts live here instead of the repository
root. They are useful context, but they are not active model or calibration
state.

- `claude_capability_test_2026-06-09/`: microfolder for testing a new Claude
  model on the current theory artifact, active Python model, calibration system,
  and project organization.
- `intergen_paper_expansion_handoff_20260708.md`: handoff prompt for expanding
  the current paper draft with the quantitative model, solution, calibration,
  diagnostics, and policy sections while fixing the main correctness issues.
- `intergen_full_calibration_audit_claude_code_20260713.md`: read-only,
  adversarial Claude Code audit of the repaired production calibration, relaxed
  search, focused `h_bar_0`-by-`chi` profile, identification, numerical
  robustness, and full target/parameter contract.
- `intergen_bequest_exit_battery_codex_20260714.md`: proper joint SMM battery
  for the parent-gated bequest and reduced-form old-age owner borrowing margin,
  including external variants, hard ownership-path acceptance, a
  nuisance-reoptimized `theta0` profile, and a fresh winner Jacobian.
- `intergen_bequest_identification_handoff_20260715.md`: new-chat handoff that
  treats bequests as a required model component and redirects the next stage to
  repairing and expanding wealth/estate identification rather than using
  old-age ownership or a near-zero coefficient as a deletion rule.
