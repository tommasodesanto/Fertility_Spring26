Run Workflow

1. The one-file runner

run_intergen_model.py is deliberately short and legible. Its top constants define the source record, output folder, target set, grid, equilibrium iteration cap, and whether to run optional proof-of-concept policy cases. It then builds a subprocess command against tools/build_intergen_mechanics_packet.py.

The runner prints the start time, source theta path, output folder, target set, Python interpreter, grid, and max_iter_eq. At completion it prints a runtime and the first files to open.

2. The mechanics packet builder

build_intergen_mechanics_packet.py loads theta from JSON or CSV, resolves the grid, fetches targets and weights with get_target_set, solves the model with run_model_cp_dt, extracts moments, computes the diagnostic loss, and writes plots/tables.

The default source is the June 23 saved current/balanced diagnostic theta: output/model/intergen_room_distribution_current_best_20260623/summary.json.

3. The active target set

The one-file runner uses candidate_replacement_young_old_roomgap_v1. In calibration.py, this target set extends the candidate replacement system with young ownership, old ownership/retention objects, and the childless owner-renter mean room gap.

Math Objects

Equations below are high-level and schematic. The implemented model is a discretized Bellman problem with Markov income, child-state transitions, Type-I-EV smoothing over discrete choices, and numerical interpolation. Use the code for exact feasibility screens and transition order.
State space and timing

A household state can be summarized schematically as s = (b, ten, i, j, z, n, c_s), where b is liquid wealth, ten is current tenure/rung, i is market/location. In the active strand I=1, so location is mechanically present but economically degenerate. j is age period, z is Markov income state, n is completed fertility/parity, and c_s is child stage.

Lifecycle Decision periods are four years: age 22, 26, ..., 82, with J=16. Fertility choices are available over the configured fertile window, roughly ages 26 to 42.
Preferences and household problem

The exact utility terms are implemented in the Bellman code and kernels. At a high level, the per-period payoff trades off consumption, housing services, fertility/child states, tenure frictions, and bequest motives.

Schematic Bellman V_j(s) = max over savings, housing/tenure, fertility, and market choices of u(c, h; n, c_s) + fertility terms + discrete-choice smoothing + beta E[V_{j+1}(s') | s, choices].

Fertility remains a one-shot completed-family choice for childless fertile households. It is not a sequential second-birth hazard model. The reported tfr uses the project convention tfr = 2 * mean_completed_fertility.

Housing and tenure

Renters choose continuous housing services up to hR_max. Owners choose from the discrete ladder H_own. Active current run: [2, 4, 6, 8, 10].

Renter object, schematic h_R in [0, hR_max], with flow housing cost proportional to the user cost/rent object.
Owner object, schematic h_O in H_own, owner flow cost includes depreciation/property-tax style costs and a financed-share constraint. The owner premium chi scales residual owner services after the physical family-space floor.

The down-payment screen uses phi as the financed share. Therefore the down-payment threshold is (1 - phi) P h, not phi P h.

The default model leaves the payment-to-income screen off. If manually enabled, it blocks purchases or adjustments only when the simplified interest-plus-tax payment on actual transaction debt exceeds pti_limit * y; cash-rich households can relax that screen by borrowing less. Incumbent owners keeping the same rung do not requalify each period.

Income, children, and transitions

The active one-file runner uses five Markov income states through income_process_overrides(5). Income is age-specific and multiplied by the current income state during working ages; retirement income is the common pension flow in the current simplification.

Markov income z' ~ Pi_z(z' | z), and y_{j,z} = z * y_j for working-age states.

Children age through a single dependent-child stage with geometric maturation calibrated to an expected 18-year child-at-home duration under four-year decision periods.

Market clearing and pricing

The active strand has one aggregate housing-services market. The code clears housing demand against an upward-sloping supply curve and uses a user-cost relationship in stationarity.

User cost relation q = (r + delta + tau^p) P.
Supply relation H^S(P) = H0 * (q / q_bar)^eta. The implemented one-market update computes a target price from current aggregate housing demand and supply primitives.

In solve_markov_income_equilibrium, the code iterates prices, tracks the best market residual, and then applies a scalar one-market refinement when enabled.

Objective and diagnostic moments

The packet computes a diagnostic squared-error loss over the active target set:

Diagnostic loss L(theta) = sum_m w_m [moment_m(theta) - target_m]^2, plus a penalty if the market residual is missing or too large.

This is used for ranking/diagnosis. The current run is not a production SMM calibration, and results should be interpreted as mechanism diagnostics.

Warnings And Gotchas

The active one-market intergenerational strand is current for this June 2026 no-location model, but it is not a final production calibration.
  • Do not compare losses across different target systems, geographies, room-unit normalizations, or objective definitions without verifying comparability.
  • Do not launch blind long global searches from this guide. The status notes warn that the one-market model has target-definition and mechanism tensions that need diagnosis.
  • Never underidentify SMM: if a hard target is removed, replace the identifying information, fix the parameter externally, or state underidentification.
  • parity_progression_1to2 should remain diagnostic-only unless the active fertility architecture and empirical measurement are changed consistently.
  • The second-birth housing response target disciplines housing demand, not a sequential second-birth fertility hazard.
  • phi is the financed share. The down-payment threshold uses (1 - phi).
  • Conditional renter policies are not the same as realized active housing after the tenure choice.
  • The current model drops location. Do not describe this guide as the older center-periphery spatial model.
  • The implemented estate-tax and policy cases are diagnostic proof-of-concept objects, not full policy reforms.
  • When discussing results, include target values next to model moments, not just loss values.
